# Audit de nuit n°2 — 2026-07-19

Deuxième passe nocturne, ciblée sur les surfaces jamais auditées : la chaîne
d'outillage node (`tools/`), l'accumulateur footprint, le harnais de parité,
le `.cmd` planifié, plus une re-vérification des zones déjà durcies.
4 agents adversariaux → **34 constats**, dont 3 critiques — tous dans
`tools/`, aucun dans le moteur temps réel.

## Verdict en une ligne

Le moteur navigateur était sain ; c'est le **pipeline de régénération
quotidien** qui pouvait détruire des données (statuts "I" écrasés, archives
tronquées par un kill, courses entre processus). Tout est corrigé, prouvé,
et les archives ont été régénérées avant la tâche planifiée de 17:00.

---

## Critiques (3) — tous dans tools/

### 1. `regen-archive.js` détruisait les statuts "I" à chaque run
`statusChar` écrivait tout statut non-actif comme `"T"` : les 23 lignes "I"
de BTC (11 pour ETH) seraient redevenues des "retests réussis" au prochain
regen — **vérifié sur les données réelles**. Une cassure comptée comme un
retest servi pollue aussi `approachAtr`. Corrigé des deux côtés :
`toRuntime` lit `"I"` → `INVALIDATED` (terminal, préservé) et `statusChar`
mappe A/I/T fidèlement. Compteurs A/T/I loggés à chaque écriture.

### 2. Écritures d'archives ni atomiques ni verrouillées
`regen-archive` + les 3 backfills faisaient chacun un read-modify-write
complet du même JSON : un kill en plein write laissait un fichier tronqué
(navigateur ET runs suivants cassés) ; deux processus concurrents = lost
update silencieux. Nouveau `tools/lock.js` : verrou inter-process par
`mkdir` (`poi/.archive.lock`, vol si plus vieux que 30 min, gitignoré) +
`writeArchiveAtomic` (tmp + rename). Branché sur les 4 écrivains.

### 3. `backfill-invalidation` absent du cron + bougie ouverte évaluée
Le `.cmd` quotidien ne lançait jamais l'invalidation : les zombies
s'accumulaient chaque jour jusqu'au prochain run manuel. Et le backfill
évaluait la bougie M15 **en cours** — un spike intrabar pouvait écrire un
"I" irréversible sur une clôture non finale. Le `.cmd` réécrit enchaîne
regen → invalidation → approach par symbole, loggue l'errorlevel de chaque
étape et sort avec le pire code ; le backfill saute toute kline dont le
`closeTime` est dans le futur.

## Moteur POI (7) — durcissements fail-closed uniquement, gel re-prouvé

- `poi-detector` : garde anti-lookahead stricte (`complete !== true` ou
  `now < availableAt` → `[]`, plus de coercition laxiste) ; la gate
  `minHistoryCandles` déclarée mais jamais appliquée est désormais réelle.
- `poi-config` : un override NaN sur une gate numérique (`seedImbalance`…)
  désactivait la gate en silence (`x < NaN` === false) — TypeError immédiat.
- `footprint-m15` :
  - plausibilité des timestamps de trade (rejet < 2019 ou > now+5 min) — un
    passage silencieux de Binance aux microsecondes ouvrirait un bucket par
    trade ;
  - epsilon sur `priceBin` (frontière de bin en flottant) ;
  - bornes `closed[]`/`seenTradeIds` immunisées contre un `historyCandles`
    non fini (fuite mémoire sinon) ;
  - snapshot bootcache **v3 avec empreinte** `binSize`/`timeframeMs` — un
    cache binné sur une autre grille est refusé au restore (reseed REST,
    coût unique), les v1/v2 sans empreinte aussi.

**Preuve de gel** : harnais de parité re-exécuté après toutes les
modifications — 9/11 scores identiques, mêmes 2 fenêtres divergentes
(écarts de données sub-ppm en bord de fenêtre) qu'à la baseline. Matrice
lifecycle : **17/17**.

## Accumulateur & feature (6)

- Flush borné par `poiLastMessageAt` : l'intervalle ne finalise plus un
  bucket sur la seule horloge murale pendant un silence du flux — même
  sémantique que le chemin ingest (un trade postérieur à `endTs` fait foi).
- `parity-harness` : un échantillon **vide** sortait en succès (0 cas testé
  = feu vert de gel) — désormais exit 1.
- Cache CSV Binance Vision : marqueur `.ok` par jour — un unzip interrompu
  ne laisse plus un CSV partiel réutilisé pour toujours ; zip purgé sur
  échec ; garde de plausibilité des timestamps (protection migration µs).
- Dédoublonnage du join regen (Set des `createdTs` existants).

## Rendu, liq-flux, G-Bot (6)

- `poi-render` : cull aligné sur le dessin (POI mort sans champ de touche
  dessiné jusqu'à maintenant mais cullé à sa naissance → disparition au
  scroll) ; saturation de la colonne de chips — plus d'empilement illisible
  au bas du pane, un chip qui chevaucherait est supprimé.
- `liq-flux` : la media query <860px masque le panneau en CSS sans passer
  par le toggle — rAF et visuels tournaient pour un panneau invisible.
  Gate `panelShown()` = toggle ET `offsetParent`.
- G-Bot : à la **reconnexion WS**, la dernière bougie tronquée pendant la
  coupure n'était réparée qu'au prochain trade tardif — `fillGap`/
  `fillGapSec` déclenché dès `onopen`.
