# Audit 6 — 2026-07-22 (4 auditeurs parallèles + vérification croisée)

Périmètre : la totalité du code servi — `index.html`, `sw.js`, les 11 modules `poi/*.js`.
Méthode : 4 auditeurs indépendants (cœur chart / moteur POI / périphériques+SW / transversal
perf-réseau-stockage-sécu), chaque finding auto-réfuté avant retenue, puis vérifié dans le code
avant application. **17 correctifs appliqués** (tous « sûrs » : petits, sans changement de
sémantique utilisateur), **4 points risqués documentés** ci-dessous pour décision.

## Correctifs appliqués

### Service worker (le finding le plus important — confirmé par 3 auditeurs indépendants)
1. **sw.js** — le repli `ignoreSearch` servait la **plus vieille** version jamais encachée
   (ordre d'insertion du Cache API) : en panne GitHub, index frais + modules antiques =
   app Frankenstein au moment exact où le bouclier devait servir. → match **exact d'abord**,
   `ignoreSearch` en secours seulement.
2. **sw.js** — `cache.put` n'écrase que l'URL exacte (query comprise) : chaque bump `?v=NN`
   ajoutait une entrée, croissance **sans borne**. → purge des variantes du même chemin
   (`cache.delete(req, {ignoreSearch:true})`) avant chaque put : 1 entrée par fichier.
3. **sw.js** — `/G-Bot/` et `/GON-TV/` vivent sur le MÊME origin : le nom fixe `gon-shell-v1`
   partageait le cache entre les deux apps (quota commun, une purge de l'une aurait détruit
   le bouclier de l'autre). → nom dérivé du **scope** (fichiers toujours byte-identiques).
4. **index.html** — enregistrement du SW restreint à `github.io` (en dev local, un shell en
   cache masquerait le code frais).
5. **sw.js** — doc corrigée : le bouclier est **total à la 2e visite** (le SW ne contrôle pas
   la visite qui l'installe).

### Rendu POI
6. **poi-render.js `anchorSec`** — le cache d'ancres mémorisait des ancres calculées sur une
   fenêtre M15 **incomplète** (bougie courante en formation, replay tronqué, scroll-back pas
   encore chargé) et ne les recalculait jamais. → on ne mémorise que si la fenêtre est
   entièrement couverte par les bougies chargées.
7. **poi-render.js `fmtPrice`** — 2 décimales rendaient tous les chips identiques (« 0.01 »)
   sur les symboles sub-dollar (1000PEPE, DOGE, SHIB…). → 4 chiffres significatifs sous 1.

### Moteur POI / réseau
8. **poi-feature.js** — `render.repaint()` inconditionnel forçait un paint complet (~10k POI
   filtrés + tris + lasers) **chaque seconde** même à vue figée. → repaint seulement sur
   bucket clos (les mutations réelles marquent déjà le rendu).
9. **poi-feature.js `recoverPoiGap`** — jusqu'à 50 pages aggTrades (poids 1000) **sans pause**
   au réveil machine, en simultané avec fillGap et les reconnexions. → 75 ms entre pages
   (même cadence que le bootstrap).
10. **poi-feature.js** — en panne GitHub, le SW sert la dernière archive encachée **sans
    signal** ; si elle date, le trou de couverture (cutoff périmé → fenêtre bootstrap) était
    invisible. → `console.warn` si l'archive a plus de 24 h.
11. **poi-lifecycle.js** — `touchCount` recomptait une bougie déjà comptée quand un re-seed
    rejouait une bougie antérieure au pointeur (2 sites). → gate `>=` (statuts/pénétration
    étaient déjà idempotents ; seul le compteur dérivait).

### Cœur chart (index.html)
12. **checkAlerts / fillGap** — les pushes de `fillGap` désalignaient `indTrs` si le WS tiquait
    entre deux rounds (ATR faux transitoire). → `indStale` posé au premier push du round.
13. **startWs → startPoll** — le chemin « 2 connexions mortes » laissait `staleTimer` orphelin.
    → clear avant bascule.
14. **Cap MAX_BARS** — `oldestDone` restait `true` après rasage de la tête (scroll-back gelé
    à tort) sur les klines. → remis à `false` (2 sites : applyBar + fillGap).
15. **persistBars quota** — l'éviction prenait `keys[0]` (ordre d'énumération arbitraire) et
    pouvait supprimer le contexte le plus frais. → éviction du plus ancien par horodatage
    (entrée corrompue prioritaire).
16. **Rattrapage from-cache** — silencieusement sauté si un fillGap de l'ancien contexte
    détenait encore `gapFilling` (commutation A→B→A rapide). → re-tentative 250 ms gardée
    par `seq`.

### Périphériques
17. **liq-flux.js** — les boules d'ambiance utilisaient les dimensions **bitmap** (×dpr) au
    lieu des px CSS : spawn hors cadre en rétina. → `fluxW/fluxH`. Plus : fade/purge élargis
    de +1 px (liseré fantôme d'arrondi DPR fractionnaire), heures du journal en **locale**
    (l'axe du chart l'est) — idem whale-sonar.
18. **confluence.js** — refiltrait ~10k POI **par frame à 60 fps** → mémoïsé sur l'identité du
    tableau (réassigné à chaque mutation). Plus : gestion **429/418 + Retry-After** (marteler
    entretient les bans IP Binance) et `AbortSignal.timeout(10s)` sur le fetch klines.

## Points RISQUÉS — à trancher par Meddy (rien appliqué)

- **R1 (important) — Alertes de prix : les mèches sont ratées en mode polling.** `checkAlerts`
  ne voit que le `close` échantillonné (2 s) : un spike qui traverse le niveau entre deux ticks
  ne sonne pas — précisément dans le mode de secours utilisé quand le WS est muet. Correctif
  proposé : tester aussi `high/low` du bar dans le chemin polling. Change la sémantique (plus
  d'alertes tireront) → décision produit.
- **R2 — Rattrapage TF secondes < 1000 trades traité comme « live »** : sur symbole illiquide,
  une veille peut rejouer des alertes rétroactives. Correctif : seuil par ÂGE du dernier trade
  (~10 s) plutôt que par taille de page.
- **R3 — Extrapolation mensuelle** : `lOf/tOf` extrapolent à 30 j fixes hors données alors que
  la grille 1M est calendaire (28-31 j) → dérive sous-bougie des ancres posées dans la marge.
  Impact minime ; le fix touche le cœur du mapping temps↔index.
- **R4 — Perf en mode FORT très dézoomé + élite pulsante** : ~1000 fantômes + ~2000 traits
  repeints à 30 fps. Si jank constaté : plafonner le glow au-delà de N traits (comme les ✦).

## Vérifié sain (auto-réfutations abouties — ne pas re-chasser)
Courses TF/symbole (loadSeq/ctxFresh étanches), curseurs aggTrades par id (ni perte ni
double-compte), anti-alertes rétroactives klines (gapFilling), anti-boucle du watcher reload
(12 min), XSS (esc/whitelists/CSP+SRI corrects, aucun innerHTML de donnée externe), collisions
d'ids archive↔live (dédup voulue), fuites rAF/listeners/sockets (aucune), TF composés
non-diviseurs (grilles cohérentes), localStorage corrompu (whitelist DKEYS + validation
profonde), cross-serving SW entre /G-Bot/ et /GON-TV/ (impossible : chemins distincts).

## Contrainte d'exploitation
**Ne jamais laisser les deux déploiements diverger** : même origin ⇒ localStorage partagé
(`orion-*`, `gon.*`). Deux versions différentes de `BARS_SCHEMA` ou du format bootcache se
supprimeraient mutuellement les données à chaque écriture. Le miroir byte-identique est la
protection — la garder.
